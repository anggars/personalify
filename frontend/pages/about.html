About Personalify

The project scoop, from exam brief to deployment.

Just a Side-Quest?

Alright, so here's the plot twist. Personalify basically wasn't just a random side-quest I built for fun. This whole project was actually my final exam for my 'Distributed Data Processing' class back in semester 6. The brief was to build a system that integrates several database technologies. We had to pick a use case, so I went straight for the 'Streaming Service Metadata Platform', and this web is the final result. It started as an assignment, but evolved into a real passion project.

This whole thing is built with a Python (FastAPI) backend, seamlessly pulling data from the Spotify Developer API. While local development relies on Docker containers, production runs purely as a Serverless app on Vercel. Main data lives in Neon (Serverless Postgres), sync history goes to MongoDB Atlas, and Upstash Redis handles the cache. Lyrics are retrieved via a custom proxy routing strategy to bypass Genius restrictions, and the vibe check? That's a Hugging Face NLP model.
